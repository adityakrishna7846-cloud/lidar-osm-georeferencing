# MATLAB Scripts

This directory contains the MATLAB scripts developed during the investigation of LiDAR, GPS, and OpenStreetMap (OSM) alignment for autonomous vehicle mapping.

The scripts are intended to be executed sequentially, with each script representing a specific stage of the georeferencing workflow.

---

## Workflow

```
Raw LiDAR Point Cloud (.pcd)
            │
            ▼
01_align_gps_to_lidar.m
            │
            ▼
Aligned LiDAR Point Cloud
            │
            ▼
02_extract_osm_local.m
            │
            ▼
Extracted OSM Local Coordinates
            │
            ▼
03_compare_osm_local.m
            │
            ▼
Local Coordinate Comparison
            │
            ▼
04_compare_osm_latlon.m
            │
            ▼
Global Coordinate Comparison
```

---

## Script Descriptions

### 01_align_gps_to_lidar.m

**Objective**

Align the LiDAR point cloud with the recorded GPS trajectory.

**Operations**

* Load LiDAR point cloud
* Load GPS trajectory
* Convert GPS coordinates from WGS84 to ENU
* Estimate heading difference
* Rotate LiDAR map
* Translate LiDAR map
* Verify alignment using 2D and 3D visualizations

**Inputs**

* `fixed.pcd`
* `trajectory_xyzyaw.csv`

**Outputs**

* Aligned LiDAR point cloud
* Alignment transformation
* Verification plots

---

### 02_extract_osm_local.m

**Objective**

Extract the local Cartesian coordinates embedded within the Lanelet2/OpenStreetMap file.

**Operations**

* Read OSM XML
* Extract `local_x`
* Extract `local_y`
* Convert XML attributes to numeric coordinates

**Inputs**

* `new_lanelet2_maps.osm`

**Outputs**

* Local OSM coordinate arrays

---

### 03_compare_osm_local.m

**Objective**

Compare the local OSM coordinate system with the aligned LiDAR point cloud.

**Operations**

* Load extracted local coordinates
* Transform coordinate axes to match the LiDAR frame
* Overlay OSM and LiDAR datasets
* Verify local alignment

**Inputs**

* Extracted OSM local coordinates
* Aligned LiDAR point cloud

**Outputs**

* OSM–LiDAR comparison plot
* OSM centerline point cloud

---

### 04_compare_osm_latlon.m

**Objective**

Compare the global geographic coordinates stored within the OSM file against the recorded GPS trajectory.

**Operations**

* Extract OSM latitude and longitude
* Convert both GPS and OSM coordinates into a common ENU frame
* Compare global positions
* Measure translation offsets

**Inputs**

* `trajectory_xyzyaw.csv`
* `new_lanelet2_maps.osm`

**Outputs**

* GPS vs OSM comparison plot
* Estimated global offset statistics

---

## Notes

* GPS coordinates are referenced using the WGS84 geodetic coordinate system.
* Local Cartesian coordinates are represented in meters.
* The LiDAR point cloud and OSM local coordinates require coordinate frame transformations before meaningful comparison.
* This workflow documents the investigation performed to understand the relationship between LiDAR, GPS, and OSM coordinate systems generated by Vector Map Builder.
